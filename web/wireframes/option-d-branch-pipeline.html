Option D: Branch Pipeline

A horizontal pipeline visualization shows the promotion flow: you → branch → staging → main. Each stage is clickable to show the diff between that stage and the next.

Concept: Think CI/CD pipeline visualization. Each node shows a count of changes at that stage. Clicking a node shows the diff between it and the upstream stage. The pipeline replaces the status bar and integrates naturally with the git-based workflow.

Pros
  • Makes the promotion flow visually obvious
  • At-a-glance counts for every stage
  • Clicking a stage gives focused comparison
  • Maps directly to git workflow mental model
  • Comparison panel is optional — collapses away
  • Natural place for "promote" / "merge" buttons later
Cons
  • Pipeline + comparison takes ~120px vertical space
  • Users unfamiliar with CI/CD may not recognise the metaphor
  • Requires fetching comparison data for each stage on-click
  • More complex state management (which stage is selected)
