Option A: Layered Status Bar

A thin bar below the edit toolbar shows three change layers as clickable segments. Each segment expands to show its changes. Concept tree nodes get multi-dot indicators.

Concept: Edit mode is always on when authenticated — no enter/exit toggle. The toolbar shows mode switcher, undo/redo, changes, history, and save. The status bar sits between the toolbar and content; each layer is a clickable pill that expands a flyout with the change list for that layer.

Pros
  • Always visible — glanceable counts for all 3 layers
  • Familiar pattern (like git status bar in VS Code)
  • Flyout keeps detail on demand without leaving context
  • Coloured dots on tree give instant visual mapping
  • No enter/exit friction — editing is always available
Cons
  • Takes vertical space — another bar below toolbar
  • Three numbers can be confusing if layers aren't understood
  • Dot indicators on tree may clash with existing warning badges
  • Flyout overlaps content when open
