Change Visibility: Design Options

How to show what's changed at each layer: local edits, branch, staging, main

The Problem

When editing a vocabulary, the user needs to understand three layers of changes:

  1. Local (unsaved) — edits in memory, not yet committed to the branch
  2. Branch vs Staging — commits on edit/staging/vocab not yet merged into staging
  3. Staging vs Main — changes in staging not yet promoted to main

Options

Option Approach Where Disruption
A: Layered Status Bar Thin bar below toolbar with 3 clickable layer pills + flyout Scheme page Medium
B: Tabbed Changes Popover Extend existing changes popover with Local/Branch/Staging tabs Scheme page Low
C: Change Drawer Right-side panel showing all layers stacked with flow arrows Scheme page High
D: Branch Pipeline Horizontal pipeline (main ← staging ← branch ← you) Scheme page Medium
E: Workspace Overview Vocab cards with change bars and expandable layer details Workspace page Low (additive)

Recommendation

These aren't mutually exclusive. A strong combination would be:

  • B or D for the scheme page (in-edit visibility)
  • E for the workspace page (pre-edit overview)

B is lowest disruption (extends existing popover), while D makes the git flow more explicit. Both need the same underlying data: branch comparison via GitHub API.
